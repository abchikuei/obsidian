|   # | Tool            | What it actually does                                                                          | Common use case                                                         | Command example                             |
| --: | --------------- | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- | ------------------------------------------- |
|   1 | `ssh`           | Securely logs into another machine over the network and lets you control it from your terminal | Connect to your VPS / production server                                 | `ssh ubuntu@your-server-ip`                 |
|   2 | `nginx`         | A high-performance web server and reverse proxy that sits in front of your app                 | Serve static files, HTTPS, proxy Node.js app from port `3000` to domain | `sudo systemctl restart nginx`              |
|   3 | `curl`          | Sends HTTP requests from terminal and shows the response                                       | Test REST APIs, health checks, webhooks                                 | `curl http://localhost:3000/api/health`     |
|   4 | `nmap`          | Scans hosts and ports to discover what services are running                                    | Check whether ports `22`, `80`, `443`, `3000` are open                  | `nmap your-domain.com`                      |
|   5 | `dig`           | Queries DNS records directly from DNS servers                                                  | Debug domain issues, inspect `A`, `CNAME`, `MX` records                 | `dig example.com`                           |
|   6 | `chmod`         | Changes file permissions (who can read, write, execute)                                        | Make a script executable                                                | `chmod +x deploy.sh`                        |
|   7 | `chown`         | Changes which user/group owns a file or folder                                                 | Fix permission errors after deploy                                      | `sudo chown -R ubuntu:ubuntu /var/www/app`  |
|   8 | `openssl`       | Generates SSL certificates, keys, hashes, and inspects HTTPS certs                             | Create private keys, inspect TLS cert expiration                        | `openssl x509 -in cert.pem -text`           |
|   9 | `host`          | Fast DNS lookup command with simple output                                                     | Quick domain ŌåÆ IP lookup                                                | `host google.com`                           |
|  10 | `route`         | Shows and edits network routing rules                                                          | Debug why traffic is not reaching gateway                               | `route -n`                                  |
|  11 | `apache`        | Traditional web server similar to nginx                                                        | Host PHP sites, static websites, reverse proxy apps                     | `sudo systemctl status apache2`             |
|  12 | `sudo`          | Runs commands with administrator privileges                                                    | Install packages, restart services                                      | `sudo apt install nginx`                    |
|  13 | `systemctl`     | Starts, stops, restarts, enables Linux services                                                | Restart backend service after deployment                                | `sudo systemctl restart myapp`              |
|  14 | `htop`          | Interactive system monitor for processes, CPU, memory                                          | Find which process is eating RAM / CPU                                  | `htop`                                      |
|  15 | `du`            | Shows how much space files/folders consume                                                     | Find huge folders in server                                             | `du -sh /var/log/*`                         |
|  16 | `df`            | Shows total disk usage and free space                                                          | Check if server disk is full                                            | `df -h`                                     |
|  17 | `wget`          | Downloads files from URLs                                                                      | Download binaries, backups, releases                                    | `wget https://example.com/app.tar.gz`       |
|  18 | `ping`          | Checks whether a host is reachable and measures latency                                        | Check if server/domain is alive                                         | `ping google.com`                           |
|  19 | `telnet`        | Opens raw connection to a remote port                                                          | Test whether a port is open                                             | `telnet example.com 443`                    |
|  20 | `traceroute`    | Shows every network hop between you and target                                                 | Diagnose slow routing path                                              | `traceroute google.com`                     |
|  21 | `tracepath`     | Similar to traceroute without root privileges                                                  | Same routing diagnostics on limited systems                             | `tracepath google.com`                      |
|  22 | `netstat`       | Lists listening ports and active network connections                                           | Check what app is bound to `3000`                                       | `netstat -tuln`                             |
|  23 | `nslookup`      | Another DNS inspection tool                                                                    | Verify DNS propagation                                                  | `nslookup example.com`                      |
|  24 | `ifconfig`      | Shows network interface configuration                                                          | Check local IP address / interfaces                                     | `ifconfig`                                  |
|  25 | `whois`         | Shows domain registration and ownership metadata                                               | Inspect domain registrar, expiry date                                   | `whois example.com`                         |
|  26 | `whoami`        | Shows the current logged-in user                                                               | Confirm current shell user                                              | `whoami`                                    |
|  27 | `netcat` (`nc`) | Swiss-army knife for TCP/UDP sockets                                                           | Test ports, send raw HTTP, create listeners                             | `nc -zv localhost 3000`                     |
|  28 | `rsync`         | Efficiently syncs files between machines                                                       | Deploy project files to server                                          | `rsync -av ./dist user@server:/var/www/app` |
|  29 | `lsof`          | Shows which process is using which file/port                                                   | Find process using port `3000`                                          | `lsof -i :3000`                             |
|  30 | `tcpdump`       | Captures raw network packets                                                                   | Debug incoming requests, DNS issues                                     | `sudo tcpdump -i any port 443`              |
|  31 | `ngrok`         | Exposes local server to public internet through a tunnel                                       | Test webhooks from Stripe / Telegram / GitHub                           | `ngrok http 3000`                           |
|  32 | `scp`           | Securely copies files over SSH                                                                 | Upload `.env`, backups, builds                                          | `scp .env user@server:/var/www/app`         |
